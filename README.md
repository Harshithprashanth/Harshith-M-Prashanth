## 🧑‍💻 About Me

I am a passionate **Research & Development engineer** with a strong foundation in **Electronics and Communication Engineering (B.E. ECE)** and currently advancing my expertise through an **M.Tech in Data Science**. My academic and professional journey sits at a rare and powerful intersection — hardware-level signal intelligence, rigorous data science methodology, and cutting-edge **Large Language Model (LLM) engineering**.

My core focus is on **LLM research and LLMOps** — designing, fine-tuning, and deploying large language models for real-world, production-grade applications. I have hands-on experience building end-to-end **LLM pipelines**, including **Retrieval-Augmented Generation (RAG)** systems, **QLoRA/LoRA fine-tuning** workflows for domain adaptation, **multi-agent orchestration frameworks**, and **LLM evaluation suites** using RAGAS, DeepEval, and TruLens.

What sets me apart is my unique background: an ECE engineer's intuition for signal processing and embedded constraints, combined with a data scientist's analytical rigour and an LLM engineer's production-deployment mindset. I bring this triple perspective to problems in **industrial IoT**, **biomedical signal analysis**, **Edge AI**, and **scientific research automation**.

My research vision is to advance the frontier of **domain-adaptive LLMs** — models that understand physical signals, engineering contexts, and scientific literature — while solving the hard engineering problems of making them efficient, reliable, and deployable on resource-constrained hardware.

I am actively seeking **research collaborations**, **PhD opportunities**, and **R&D roles** where I can contribute to the next generation of intelligent, LLM-powered systems.

## 🚀 Featured Projects

### ━━━━ LLM & LLMOps Projects ━━━━

<details>
<summary>🤖 1. Domain-Adaptive RAG System for Scientific Literature</summary>

**Domain:** LLMOps · RAG · NLP

**Description:**
A production-grade Retrieval-Augmented Generation system designed for scientific and engineering literature. The pipeline ingests PDFs and arXiv papers, performs intelligent chunking and embedding using `sentence-transformers`, stores vectors in ChromaDB, and serves queries via a fine-tuned LLM backend. Advanced retrieval techniques include **HyDE** (Hypothetical Document Embeddings), **cross-encoder re-ranking**, and **query decomposition**.

**RAGAS Evaluation:** Faithfulness: `0.91` | Answer Relevancy: `0.88`

**Tech Stack:**
`Python` `LangChain` `ChromaDB` `Sentence-Transformers` `OpenAI API` `HuggingFace` `RAGAS` `FastAPI` `Docker`  

</details>

<details>
<summary>⚙️ 2. LLM Fine-tuning Pipeline for Technical Domain Adaptation</summary>

**Domain:** LLMOps · LLM Fine-tuning

**Description:**
End-to-end **QLoRA fine-tuning pipeline** for Llama-2 and Mistral-7B on engineering domain corpora. Leverages 4-bit quantization via BitsAndBytes for a **65% GPU memory reduction** while retaining **97% of full fine-tuning performance**. Pipeline includes automated dataset preparation, instruction formatting (Alpaca/ChatML templates), gradient checkpointing, and GGUF conversion for Ollama deployment.

**Results:** 65% VRAM reduction · 97% performance retained · 3.2× inference speedup

**Tech Stack:**
`PyTorch` `HuggingFace Transformers` `PEFT` `TRL` `BitsAndBytes` `W&B` `MLflow` `Ollama` `LangChain`  

</details>

<details>
<summary>🕸️ 3. Multi-Agent LLM Orchestration Framework</summary>

**Domain:** LLMOps · Agentic AI

**Description:**
A multi-agent research automation system built with LangChain and LangGraph. Specialized agents handle literature review (RAG), data analysis (code interpreter), summarisation, citation management, and report generation. Implements **ReAct patterns** with tool integrations: arXiv API, Semantic Scholar, Python REPL. Full observability via LangSmith tracing.

**Results:** 78% reduction in manual research workflow time

**Tech Stack:**
`LangChain` `LangGraph` `LangSmith` `OpenAI API` `Claude API` `FastAPI` `Redis` `Docker`  

</details>

<details>
<summary>🌐 4. LLM-Powered IoT Anomaly Detection & NL Interface</summary>

**Domain:** LLMOps · IoT · Edge AI

**Description:**
Quantized Mistral-7B deployed via Ollama on an edge server interfaced with an industrial IoT sensor network. A RAG pipeline injects real-time sensor context, enabling operators to query system state in plain English: anomaly detection, predicted failures, and maintenance schedules. Function calling enables direct actuator control through natural language commands.

**Results:** Sub-2s response on edge hardware · 93% query accuracy

**Tech Stack:**
`Ollama` `Mistral-7B` `LangChain` `MQTT` `InfluxDB` `Grafana` `Raspberry Pi` `FastAPI` `ChromaDB`  

</details>

<details>
<summary>📊 5. LLM Evaluation & Benchmarking Suite</summary>

**Domain:** LLMOps · LLM Evaluation

Comprehensive evaluation framework comparing GPT-4, Claude, Gemini, Llama, Mistral, and Phi-3 across factual accuracy, reasoning, domain knowledge, and hallucination rates. **Tech Stack:**
`OpenAI API` `Anthropic API` `HuggingFace` `DeepEval` `RAGAS` `TruLens` `W&B` `Streamlit` `Docker`
